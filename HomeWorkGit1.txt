   
Описание задания.

 1 уровень сложности: 1. Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделано предварительно?


2. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

3. Как получить на локальный компьютер изменения с github.com

4. Какую информацию нам дают команда :
git status
git log
git remote -v 


ответы:
1)Перейдите на страницу репозитория на GitHub.com и скопируйте URL-адрес репозитория(кнопку "Code", SSH), введите команду git clone в терминале (или GitBash) через пробел вставьте скопированный ранее URL-адрес репозитория и через пробел путь к папке куда сохранить. Enter. 
Предварительно нужно, чтобы на компьютере был установлен git, чтобы был добавлен публичный ключ SSH в GitHub.
2)в терминале перейти в папку, файлы которой нужно добавить в репозиторий, выполнить команду git init для инициализации и выполнить команду git add . (Если нужно добавить все файлы из папки) или git add и через пробел имя файла (если нужно добавить один или несколько файлов). Выполнить команду git commit -m и в кавычках написать свой комментарий описания внесенных изменений, например, "Initial commit". Далее создать новый репозиторий на GitHub.com и связать локальный репозиторий с удаленным при помощи команды git remote add и через пробел вставить ссылку на репозиторий, выполнить. Далее отправить свои изменения выполнив команду git push.
3)в терминале перейти в папку локального репозитория, где нужно получить изменения, выполнить команду 
git pull, чтобы загрузить последние изменения и автоматически выполнить слияние. Или можно выполнить это в 2 команды: git fetch - загрузить изменения и git merge - чтобы выполнить слияние. Отдельно обычно нужно делать, когда происходят конфликты при слиянии.
4)git status - проверка текущего состояния локального репозитория git, показывает состояние файлов
git log - покажет историю комитов(изменений проекта)
git remote -v - покажет список удаленных репозиторием, которые связаны с моим локальным, часто используют для проверки связи с репозиторием git.
